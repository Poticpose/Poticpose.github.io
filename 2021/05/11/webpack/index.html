Webpack

Webpack 基本概念

Webpack 初次使用

public 文件下大多数为静态资源，如 index.html

src 文件下主要存放待打包程序，即资源文件，如 js、css 等

npm init -y 可以生成项目描述文件

npm i webpack webpack-cli -D 将 webpack 和 webpack-cli（即 webpack 指令） 加入到开发依赖中。

可以利用 npx webpack ./src/main.js –mode=development 对 main.js 进行打包，打包模式为开发模式

npm 用于下载包，npx 会将项目文件中的 .bin 文件临时视为环境变量，接下来就可以访问其中的应用程序。

随后在静态资源中引入打包后的文件即可在浏览器中运行。

Webpack 大核心概念

webpack 打包的时候会以一个文件或多个文件作为入口，输出一个或多个文件

Webpack 配置文件

webpack 配置文件位于项目根目录，且命名固定为 webpack.config.js

webpack 配置最终都在 node.js 平台执行，因此需要使用 commonJS 模块化规范

随后直接运行 npx webpack 即可
