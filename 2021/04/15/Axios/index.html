Axios 使用方式

get，post，通用方式在 AJAX 中已做介绍

Axios 响应

  • config：请求对象信息
  • data：请求体
  • headers：请求头
  • request：AJAX 原生请求对象（即 axios 发送请求时所创建的 AJAX 对象）
  • status：响应状态码
  • statusText：响应状态字符串

Axios 配置对象

config

  • url：设置 url 参数，明确向谁发送请求
  • method：设置请求类型
  • baseURL：设置 url 的基础结构（所发送的请求目标为 baseURL + url）
  • transformRequest：对请求进行处理，将处理结果发送至服务器
  • transformResponse：对响应进行处理，将处理结果传给回调函数进行处理
  • headers：配置请求头
  • params：配置参数
  • paramsSerializer：function(params){}：对请求参数进行序列化，将其转化为字符串
  • data：配置请求体（两种形式，对象/字符串，对象转化为 JSON 格式进行传递，字符串直接传递）
  • timeout：超时时间，超过时间请求取消
  • withCredentials：跨域请求时，对 cookie 携带进行设置，false 为不携带，true 可以在跨域时携带 cookie
  • adapter：对请求适配器进行设置，一个是发送 AJAX，另一个在 Node.js 中发送 Http 请求
  • auth：请求的基础验证，比如设置用户名和密码
  • responseType：对响应体结果的格式进行设置，如 json，默认也为 json
  • responseEncoding：响应结果的编码，如 utf-8
  • xsrfCookieName：对 cookie 的名字进行设置
  • xsrfHeaderName：对头信息进行设置
  • onUploadProgress：在上传时进行回调
  • onDownloadProgress：在下载时进行回调
  • maxContentLength：设置 HTTP 响应体的最大尺寸，单位为字节
  • maxBodyLenght：设置 HTTP 请求体最大尺寸
  • validateStatus：对响应结果的成功进行设置（什么情况下才算成功）
  • maxRedirects：最大跳转次数
  • socketPath：socket 文件位置，用于向 docker 守护进程发送请求，即数据转发
  • httpAgent：对客户端信息进行设置
  • proxy：设置代理
  • cancelToken：对 AJAX 请求进行取消的设置

Axios 拦截器

过程：请求拦截器 -> 响应拦截器 -> axios 回调处理

注意：请求拦截器在源码层面中，多个拦截器会被放置到一个数组当中。而存放请求拦截器的数组是逆序的，存放响应拦截器的数组是顺序的。即存在多个拦截器，如：请求拦截 1，请求拦截 2，响应拦截 1，响应拦截 2。在触发过程中的顺序是：请求拦截 2 -> 请求拦截 1 -> 响应拦截 1 -> 响应拦截 2
